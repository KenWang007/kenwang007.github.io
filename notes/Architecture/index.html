自动生成方式（Jekyll 环境下）

要在 Jekyll 中自动生成目录下的文件列表，可以使用以下 Liquid 模板语法：

{% assign current_dir = page.dir %}
{% assign files = site.pages | where_exp: "p", "p.dir == current_dir" %}
{% for file in files %}
{% if file.name != "index.md" %}
- [{{ file.title | default: file.name | remove: ".md" }}]({{ file.name }})
{% endif %}
{% endfor %}

将上述代码添加到 index.md 文件中，Jekyll 会在构建时自动生成目录下的所有 Markdown 文件列表。

工作原理

  1. page.dir 获取当前页面所在目录
  2. site.pages 包含所有 Jekyll 生成的页面
  3. where_exp 过滤出当前目录下的所有页面
  4. 遍历所有页面，排除 index.md 文件，生成链接列表

注意事项

  • 只有在 Jekyll 环境下（如 GitHub Pages）才会自动生成
  • 本地直接打开文件不会显示自动生成的列表
  • 每个 Markdown 文件的前置元数据中的 title 会作为链接文本，否则使用文件名
